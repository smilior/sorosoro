01 要件定義 · 01/04

プロジェクト概要

おうち掃除ログ（sorosoro）の目的・背景・成功指標・体制・スコープ外・計画を定義する。

確定メモ

本ドキュメントの内容は企画書（docs/00. 企画書/README.md）および共有プロダクトブリーフ（docs/_spec/PRODUCT_BRIEF.md）に基づき確定済み。

個人開発の B2C コンシューマアプリ。本番スタックは Next.js + Vercel + Turso + Better Auth + Google OAuth（GCP）。データはログインユーザー単位でクラウドに保存する。

スコープ外（OUT-01〜OUT-06）は AI・実装の境界線として厳守する。

アプリ概要

項目 内容
アプリ名おうち掃除ログ
プロジェクトコード名sorosoro
目的家の掃除タスク（シーツ洗い、風呂の排水口など）の実施履歴と「次の目安」を、サボりを責めない UI で管理する。記録は 1 タップで完結させる。
背景「シーツいつ洗った？」が分からず罪悪感を抱えやすい。カレンダー型・期限警告型は責めの印象が強く、継続しにくい。前回からの経過日だけをリスト型で淡々と見せる道具が必要。
想定利用者家庭で掃除の周期を把握したい個人（共働き・単身・同棲など）。スマホ中心の利用者。Google アカウントでログインする。
基盤Next.js App Router（TypeScript）/ ホスティング: Vercel / DB: Turso（libSQL）+ Drizzle ORM / 認証: Better Auth + Google OAuth（Google Cloud Console）/ ミューテーション: Server Actions / スタイル: デザイントークン + Tailwind CSS v4。モバイルファーストの会員制 Web アプリ。

スコープ外(やらないこと)

ID やらないこと 理由・補足
OUT-01世帯共有・複数人での同一タスク編集個人アカウント単位に限定する
OUT-02プッシュ通知・リマインドサボりを責めない思想と両立困難
OUT-03カレンダー表示・予定日固定コンセプト外（リスト型・経過日のみ）
OUT-04ネイティブアプリWeb モバイルファーストで代替
OUT-05オフライン完全同期（PWA 本格）後続候補
OUT-06家事全般（掃除以外）掃除タスクに集中
