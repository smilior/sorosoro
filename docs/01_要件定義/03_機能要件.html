01 要件定義 · 03/04

機能要件

おうち掃除ログ（sorosoro）が提供する機能・受け入れ基準・画面構成を定義する。

確定メモ

機能 ID は F-00〜F-08、画面 ID は S-00 ログイン / S-01 ホーム / S-02 タスク登録・編集 / S-03 履歴 / S-04 記録シート。S-00 は実ログイン画面（Google OAuth）。

受け入れ基準は各 MVP 機能の Definition of Done。Given-When-Then をすべて満たせば完成とみなす。

コア仕様: ratio ソート、状態色（warn/late は淡色のみ）、同日重複ブロック、未来日不可、固定スケジュールなし。データは Turso にユーザー単位で保存する。

機能一覧

ID 機能名 区分 概要 対応画面
F-00ログイン / ログアウト必須Better Auth + Google OAuth でサインイン。セッション Cookie を発行。ログアウトでセッションを破棄しログイン画面へS-00
F-01タスク一覧（ホーム）必須ログインユーザーのタスクカードを ratio 降順で表示。状態色（通常 / warn / late）を淡色背景で付与。未記録・経過日メタを表示S-01
F-021 タップ記録（今日）必須カード上の「今日やった」で今日を logs に追記（Server Action）。pop / flash / FLIP とトーストでフィードバック。確認ダイアログは出さないS-01
F-03過去日記録必須「今日やった」長押し、または履歴の記録シートから過去日を記録。未来日は不可S-01, S-03, S-04
F-04記録アンドゥ必須トーストの「元に戻す」で直前 1 件の記録を削除S-01, S-03, S-04
F-05タスク登録必須名前 + 周期（プリセット 7/14/30 日または 1〜365 の数値）で新規登録。user_id に紐づけて Turso に保存S-02
F-06タスク編集・削除必須名前・周期の変更。削除は履歴ごと削除のインライン二段確認（confirm ダイアログは使わない）S-02
F-07履歴画面必須サマリー（目安周期 / 前回 / 次の目安）と実施日リスト。日付選択記録導線S-03, S-04
F-08クラウド永続化必須tasks / task_logs を Turso（libSQL）にユーザー単位で保存・復元。派生値（lastDone / elapsedDays / ratio / nextDue）は永続化しない全画面（要ログイン）

受け入れ基準

機能ID ID Given(前提) When(操作) Then(期待結果)
F-00AC-00未ログインでアプリにアクセスする保護された画面（S-01 等）を開こうとするS-00 ログイン画面へ誘導される
F-00AC-00bS-00 を表示している「Google でログイン」を選び OAuth を完了するセッションが確立し、自分のタスクがあるホーム（S-01）へ遷移する。初回はサンプルタスクが投入されてもよい
F-00AC-00cログイン済みログアウトを実行するセッションが破棄され、S-00 へ戻る。他ユーザーのデータは見えない
F-01AC-01ログイン済みでタスクが 1 件以上存在するホームを表示するカードが ratio 降順（同率は elapsed 降順。未記録は elapsed 最大扱い）で並ぶ
F-01AC-02ratio > 1 のタスクと ratio ≥ 1.5 のタスクがあるホームを表示するwarn は黄系淡背景、late は赤系淡背景。警告アイコンや強い赤は出ない
F-01AC-03ログイン済みでタスクが 0 件ホームを表示する「まだタスクがありません」と ＋ からの登録誘導が表示される
F-02AC-04ホームに未記録（当日未記録）のタスクがある「今日やった」を 1 回タップする確認なしで今日が logs に入り、ボタンが「記録済み」になり、トーストと pop/flash/FLIP が起きる
F-02AC-05当日すでに記録済みのタスクがある「今日やった」を再度タップする追加されず、トースト「その日はすでに記録済みです」が表示される
F-03AC-06ホームにタスクがある「今日やった」を 480ms 以上長押しする記録シート（S-04）が開き、長押し後の click は記録されない
F-03AC-07記録シートが開いている昨日 / おととい / date 入力で未記録日を選んで記録する選択日が logs に入り、同日重複はブロックされる。未来日は選択・記録できない
F-04AC-08記録直後のアンドゥ付きトーストが表示中「元に戻す」をタップする直前に追加した記録 1 件が削除され、一覧・履歴が更新される
F-05AC-09タスク登録画面を開いている名前を入力し周期を選び保存するホームに新タスクが現れ、Turso に当該ユーザーのタスクとして永続化される
F-05AC-10タスク登録画面を開いている名前が空、または周期が不正保存ボタンが disabled のまま保存できない
F-06AC-11既存タスクの編集画面を開いている名前・周期を変更して保存するホーム・履歴に反映され、既存 logs は保持される
F-06AC-12編集画面で削除を開始した二段確認で「削除する」を選ぶタスクと履歴が消えホームへ戻る。「やめる」では削除されない
F-07AC-13記録があるタスクの履歴を開いている画面を表示するサマリー（目安 / 前回 / 次の目安「ごろ」表記）と実施日リストが表示される
F-07AC-14履歴画面を開いている「＋ 日付を選んで記録」をタップする記録シートが開き、記録後にリストへ反映される
F-08AC-15タスクまたは記録を追加・変更した別端末または再ログイン後にホームを開く同一ユーザーの Turso データが復元され、データが失われない。他ユーザーのタスクは一切表示されない

画面遷移図

遷移の要点（実装・受け入れ時の正）:

  • 未ログインで保護画面へ → S-00 ログイン。Google OAuth 完了後 → S-01 ホーム
  • S-01 の ＋ → S-02（新規）。カード本体タップ → S-03。カードの記録ボタンは履歴へ遷移しない
  • S-03 の「編集」→ S-02。戻るは開いた元（ホームまたは履歴）へ
  • S-01 長押し / S-03「日付を選んで記録」→ S-04。背景タップまたはキャンセルで閉じる
  • タスク削除後 → S-01。ログアウト → S-00
