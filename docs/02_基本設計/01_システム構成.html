システム構成

おうち掃除ログ（sorosoro）のシステム全体構成・技術スタック・環境・配信方針を定義する。menva-ai 同型スタック。

確定メモ

技術選定の正は ADR-0003。実装パスはリポジトリ root の src/。prototype/ は UI 参照用の静的モックであり本番コードではない。

全体構成

Browser（モバイルファースト）
  → Next.js App Router on Vercel
       ├ Middleware: session cookie ゲート（/home 等）
       ├ Better Auth（/api/auth/*）→ Google OAuth（GCP client id/secret）
       ├ Server Actions（tasks CRUD / record / undo / seed）
       └ Drizzle ORM → Turso libSQL
            tables: users, sessions, accounts, verifications,
                    tasks (user_id, name, cycle_days),
                    task_logs (task_id, done_date unique)
  • ホスティング — 本番は Vercel。ローカルは next dev。
  • データ — ログインユーザー単位で Turso に永続化。派生値（ratio 等）はサーバー側ドメイン層で毎回計算。
  • 認証 — Better Auth + Google OAuth（GCP）。未ログインは /login へリダイレクト。

環境構成

環境 識別 アプリ DB 用途
ローカルlocalnpm run dev（:3000）turso dev → http://127.0.0.1:8080 / local.db開発
プレビューpreviewVercel Previewリモート Turso（共有 or ブランチ DB）PR 確認
本番productionVercel Productionリモート Turso一般利用

環境変数の正はリポジトリ root の .env.example（ローカルは .env.local）。主要キー:

  • TURSO_DATABASE_URL / TURSO_AUTH_TOKEN
  • BETTER_AUTH_SECRET / BETTER_AUTH_URL / NEXT_PUBLIC_BETTER_AUTH_URL
  • GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET

Google OAuth の承認済みリダイレクト URI:

  • 開発: http://localhost:3000/api/auth/callback/google
  • 本番: https://<your-domain>/api/auth/callback/google

起動順序（ローカル）

  1. npm install
  2. ターミナル1: npm run db:dev（turso dev --db-file local.db → :8080）
  3. cp .env.example .env.local（または npm run setup:local）で secret 生成・スキーマ適用
  4. npm run db:push（未実行時）
  5. ターミナル2: npm run dev → http://localhost:3000
  6. Google ログイン → ホーム（初回は空状態。＋からタスク登録）

CI/CD

  • デプロイ — Vercel の Git 連携。main（または既定ブランチ）→ Production、PR → Preview。
  • ビルド — npm run build（Next.js）。
  • DB — スキーマ変更は Drizzle マイグレーション / db:push を運用手順に従いリモート Turso へ適用。
