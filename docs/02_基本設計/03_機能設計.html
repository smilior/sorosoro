おうち掃除ログ / sorosoro
正本: PRODUCT_BRIEF · ADR-0003
02 基本設計 · 03/07

機能設計書

おうち掃除ログの各機能を外部から見た振る舞いとして定義する。Google ログイン必須・ユーザー単位で Turso に永続化。

確定メモ

内部アルゴリズムの詳細は 03_詳細設計/01_処理設計 へ。ここでは外部仕様と受け入れ基準（Given-When-Then）のみ。実装の正は src/actions/tasks.ts と src/lib/domain/task.ts。

機能一覧

機能ID 機能名 概要 対応画面 ロール
F-00Google ログイン / ログアウトBetter Auth + Google OAuth。セッション Cookie 発行・破棄S-00未認証 / 認証済
F-01タスク一覧ホームに ratio ソート済みタスクを状態色付きで表示S-01認証済
F-021タップ記録（今日）「今日やった」で当日を task_logs に追加。達成フィードバックS-01認証済
F-03過去日記録長押しまたは履歴から記録シートで過去日を記録S-01 / S-03 / S-04認証済
F-04記録アンドゥトーストから直前1件の記録を削除共通トースト認証済
F-05タスク登録名前 + 周期プリセット/数値で新規タスクS-02認証済
F-06タスク編集・削除名前・周期変更。削除は履歴ごと二段確認S-02認証済
F-07履歴画面サマリーと実施日リスト。日付選択記録導線S-03認証済

共通ルール（ratio・ソート・状態）

  • lastDone = logs の最新日（なければ null）
  • elapsedDays = today − lastDone（日付単位。未記録は null）
  • ratio = logs 空 → 1.0 / それ以外 → elapsedDays / cycleDays
  • nextDue = lastDone + cycleDays（未記録は null）
  • 並び順: ratio 降順 → 同率は elapsedDays 降順（未記録の elapsed は最大扱い）→ 同率は名前 ja localeCompare
  • 状態: ratio ≥ 1.5 → late / ratio > 1 → warn / それ以外 → normal。警告アイコン・強い赤は禁止
  • 記録: 同日重複はブロック。未来日は不可。固定スケジュール・未消化溜まり表現はしない
  • 所有: すべてのタスク/ログ操作は session.user.id === tasks.user_id に限定

外部仕様

F-00 Google ログイン / ログアウト

  • 目的: 個人データをクラウドに紐づけ、機種変更後も継続利用できるようにする
  • 対応画面: S-00 ログイン
  • 権限: 未認証はログイン可。認証済は /home へ

入力項目

項目名型必須制約・備考
OAuth 同意—必須Google アカウント。スコープ email / profile / openid

出力項目

項目名型備考
セッション CookieCookieBetter Auth が発行
遷移先URL成功時 /home（callbackUrl があれば優先）

受け入れ基準

  • Given 未ログイン When /home にアクセス Then /login へリダイレクトされる
  • Given ログイン成功 When 完了 Then 自分のタスク一覧が見える（他ユーザーのデータは見えない）
  • Given ログイン中 When ログアウト Then セッションが破棄されログイン画面に戻る

F-01 タスク一覧

  • 目的: 掃除タスクの状況を一覧で把握し、1タップ記録の入口にする
  • 対応画面: S-01 ホーム
  • 権限: 認証済ユーザー（自分のタスクのみ）

入力項目

項目名型必須制約・備考
基準日 today日付必須サーバー側当日（表示・ratio 計算用）
userId文字列必須セッションから取得

出力項目

項目名型備考
タスクカード一覧配列ratio 降順ソート済み
タスク名文字列1行省略
経過日数メタ文字列「前回から N 日・目安 M 日」または「まだ記録なし・目安 M 日」
状態背景late / warn / normalratio に応じた淡色
当日記録済み表示真偽ボタンが「記録済み」スタイル
空状態メッセージタスク0件時（シード後は通常4件）

受け入れ基準

  • Given 自分のタスクが複数ある When ホームを開く Then ratio が大きい順に並ぶ
  • Given ratio ≥ 1.5 のタスク When 表示 Then late-bg 背景で警告アイコンは出ない
  • Given 他ユーザーのタスク When 一覧取得 Then 含まれない

F-02 1タップ記録（今日）

  • 目的: 確認ダイアログなしで当日の実施を1タップ記録する
  • 対応画面: S-01
  • 権限: 認証済・タスク所有者

入力項目

項目名型必須制約・備考
taskId文字列必須自分のタスク
date日付省略時は今日未来不可

出力項目

項目名型備考
記録日string成功時 { ok: true, data: { date } }
トーストメッセージ + アンドゥ3.8 秒で自動消滅
フィードバックアニメpop / flash / FLIP 480ms

受け入れ基準

  • Given 当日未記録 When 「今日やった」をタップ Then task_logs に今日が入り、カードが再ソートで下方向へ移動しうる
  • Given 当日記録済み When 再度タップ Then 追加されず「その日はすでに記録済みです」
  • Given 記録成功 When 完了 Then 確認ダイアログは出ず、アンドゥ付きトーストが出る

F-03 過去日記録

  • 目的: 思い出した過去の実施日を記録する
  • 対応画面: S-04 記録シート（S-01 長押し 480ms / S-03 から起動）
  • 権限: 認証済・タスク所有者

入力項目

項目名型必須制約・備考
taskId文字列必須対象タスク
date日付必須今日以前。未来不可。date input の max=今日

出力項目

項目名型備考
成功ActionResultlogs に選択日が追加
トーストメッセージ + アンドゥ記録成功時
エラー文字列同日重複・未来日・未所有

受け入れ基準

  • Given ホーム When 「今日やった」を 480ms 長押し Then 記録シートが開き、発火後の click は無視される
  • Given 記録シート When 「昨日」をタップ Then 昨日が logs に入りシートが閉じる
  • Given 未来日 When 記録しようとする Then 「未来の日付は記録できません」

F-04 記録アンドゥ

  • 目的: 誤記録を気軽に取り消せるようにする
  • 対応画面: 共通トースト
  • 権限: 認証済・タスク所有者

入力項目

項目名型必須制約・備考
taskId文字列必須直前に記録したタスク
date日付必須直前に記録した日

受け入れ基準

  • Given 記録直後のトースト表示中 When 「元に戻す」をタップ Then 直前の記録1件のみ削除され一覧が更新される
  • Given トーストが 3.8 秒で消えた後 When 操作しない Then アンドゥ導線は消える

F-05 タスク登録

  • 目的: 気になる掃除を名前と目安周期で登録する
  • 対応画面: S-02（新規）
  • 権限: 認証済

入力項目

項目名型必須制約・備考
name文字列必須trim 後空不可。placeholder「例：シーツ洗い」
cycleDays整数必須1–365 に clamp。プリセット 7/14/30 または数値入力

受け入れ基準

  • Given 名前が空 When 保存 Then 登録されない（エラーまたは disabled）
  • Given 名前「シーツ洗い」周期7 When 保存 Then ホームに追加され ratio=1.0 で見える

F-06 タスク編集・削除

  • 目的: 名前・周期を後から変え、不要タスクを履歴ごと消す
  • 対応画面: S-02（編集）
  • 権限: 認証済・タスク所有者（他ユーザーの id では「タスクが見つかりません」）

入力項目

項目名型必須制約・備考
id / taskId文字列必須編集対象
name文字列必須trim 後空不可
cycleDays整数必須1–365 に clamp
削除確認二段 UI削除時confirm() は使わない。インライン確認

受け入れ基準

  • Given 編集画面 When 周期を変更して保存 Then ホームの目安表示が更新される（logs は残る）
  • Given 削除確定 When 実行 Then タスクと task_logs がカスケード削除されホームに戻る

F-07 履歴画面

  • 目的: サマリーと実施日リストを確認し、日付選択記録へ進む
  • 対応画面: S-03
  • 権限: 認証済・タスク所有者

出力項目

項目名型備考
目安の周期文字列「N日ごと」
前回文字列「N日前」等
次の目安文字列「M月D日ごろ」
実施日リスト配列新しい順。空なら空状態文言

受け入れ基準

  • Given 記録があるタスク When 履歴を開く Then サマリー3項目と日付行が表示される
  • Given 記録なし When 履歴を開く Then 「まだ記録がありません」
  • Given 履歴 When 「日付を選んで記録」 Then 記録シートが開く

F-08 クラウド永続化

  • 目的: リロード・再ログイン・機種変更後もタスク・記録を保持する
  • 対応画面: 全アプリ画面
  • 権限: 認証済（user_id スコープ）

受け入れ基準

  • Given タスク登録・記録後 When ページをリロード Then 同じデータが Turso から復元される
  • Given 初回ログインでタスク0件 When ホーム表示 Then サンプルタスクが投入されうる（seedIfEmpty）
  • Given 別ユーザーでログイン When 一覧表示 Then 先のユーザーのデータは見えない
