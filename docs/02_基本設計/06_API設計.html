API設計書

おうち掃除ログのアプリケーション API。認証は Better Auth の HTTP ルート、ドメイン操作は Server Actions。

確定メモ

実装の正は src/app/api/auth/[...all]/route.ts と src/actions/tasks.ts。公開 REST リソース API は MVP では設けない。

共通規約

  • 認証 — Server Actions は requireUserId() でセッション必須。未認証は UNAUTHORIZED → クライアント向けメッセージ「ログインが必要です」。
  • 所有 — 更新・削除・記録は tasks.user_id = session.user.id でスコープ。未所有は「タスクが見つかりません」。
  • 結果型 — ActionResult<T> = { ok: true; data: T } | { ok: false; error: string }
  • 日付 — YYYY-MM-DD。未来日の記録はエラー。
  • 再検証 — ミューテーション後 revalidatePath("/")
  • ソート・ratio — 読み取り時に deriveTask / sortTasks で計算。DB に保存しない。

Better Auth ルート

パス 概要 備考
/api/auth/*Better Auth ハンドラ全体sign-in / sign-out / session / OAuth callback 等
/api/auth/callback/googleGoogle OAuth コールバックGCP に登録するリダイレクト URI

プロバイダ: Google（GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET）。セッション Cookie 名: better-auth.session_token（Secure 環境では __Secure- 接頭）。

個別仕様

listTasksAction(): ActionResult<{ tasks: DerivedTask[]; today: string }>

  • 概要: ログインユーザーのタスクを取得し、ratio・状態を付与してソート
  • 入力: なし（userId はセッション）
  • 成功: { ok: true, data: { tasks, today } }
  • 失敗: 未認証「ログインが必要です」/ その他「タスクの取得に失敗しました」

createTaskAction({ name, cycleDays })

名前型必須説明
namestring必須trim 後空不可 → 「タスク名を入力してください」
cycleDaysnumber必須1–365 に clamp
  • 成功: { ok: true, data: { id } }
  • 失敗: 未認証 / バリデーション / 「タスクの作成に失敗しました」

updateTaskAction({ id, name, cycleDays })

  • 概要: 所有タスクの name / cycleDays のみ更新
  • 成功: { ok: true }
  • 失敗: 未認証 / 名前空 / 未所有「タスクが見つかりません」 / 更新失敗

deleteTaskAction(id)

  • 概要: 所有タスクを削除（task_logs は FK cascade）
  • 成功: { ok: true }
  • 失敗: 未認証 / 未所有「タスクが見つかりません」

recordDoneAction({ taskId, date? })

名前型必須説明
taskIdstring必須対象
datestring任意YYYY-MM-DD。省略時は today
  • 成功: { ok: true, data: { date } }
  • 失敗（error 文字列）:
    • 未認証 → 「ログインが必要です」
    • 形式不正 → 「日付の形式が不正です」
    • 未来日 → 「未来の日付は記録できません」
    • 未所有 → 「タスクが見つかりません」
    • 同日重複 → 「その日はすでに記録済みです」

undoRecordAction({ taskId, date })

  • 概要: 指定日の記録を1件削除（アンドゥ）
  • 成功: { ok: true }（該当行がなくても ok になりうる）
  • 失敗: 未認証 / 未所有「タスクが見つかりません」 / 「取り消しに失敗しました」

seedIfEmptyAction()

  • 概要: 当該ユーザーにタスクが0件のときだけサンプル4件＋過去ログを投入
  • 成功: { ok: true, data: { seeded: boolean } }
  • 既にタスクがある場合: seeded: false

エラー対応表（要約）

条件 error メッセージ例
