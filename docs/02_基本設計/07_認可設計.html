02 基本設計 · 07/07

認可設計書

おうち掃除ログの認可方針。ロールは「認証済ユーザー」のみ。データは user_id で完全分離。MVP にマルチユーザー共有はない。

確定メモ

認証: Better Auth + Google OAuth。認可ルール: すべてのタスク/ログ操作は session.user.id === tasks.user_id。未認証は /login へ。世帯共有は OUT（PRODUCT_BRIEF OUT-01）。

方針

  • 認証必須 — アプリ画面（S-01〜S-04）はセッション Cookie が必要。
  • ロールは1種 — 管理者 / ゲストの区別はない。ログインした個人が自分のデータだけ操作する。
  • データ分離 — クエリ・更新は常に user_id でフィルタ。他ユーザーの taskId を指定しても見つからない扱い。
  • 共有なし — 招待・家族アカウント・同一タスクの複数編集は MVP 外。

ロール一覧

ロール コード 説明 想定ユーザー
未認証anonymousログイン画面のみ利用可訪問者

権限マトリクス

◯=可、−=不可。

機能 / 操作 未認証 認証済（自分） 認証済（他人のデータ）
ログイン画面閲覧◯→ /home リダイレクト—
ホーム閲覧・タスク一覧−（/login へ）◯−（一覧に出ない）
記録・アンドゥ−◯−（見つからない）
タスク登録・編集・削除−◯−
履歴閲覧−◯−
シード（初回サンプル）−◯（自分の空状態のみ）−

チェック実装

層 仕組み 振る舞い
Middleware src/middleware.ts Cookie なしで /home・/app → /login?callbackUrl=...。Cookie ありで /login → /home
ページ getServerSession() /home でセッション無ければリダイレクト
Server Actions requireUserId() 無ければ Error("UNAUTHORIZED") → クライアントへ「ログインが必要です」
データアクセス eq(tasks.userId, userId) list / update / delete / record / undo すべて所有確認

セッション

  • 発行: Better Auth（Google OAuth 成功時）
  • Cookie: better-auth.session_token（本番 Secure 時は __Secure-better-auth.session_token）
  • cookieCache: 有効、maxAge 5 分（src/lib/auth.ts）
  • 信頼オリジン: BETTER_AUTH_URL / NEXT_PUBLIC_BETTER_AUTH_URL

将来メモ

世帯共有を導入する場合は、(1) リソース所有者と共有メンバーの明示モデル、(2) サーバー側での共有権限チェック、(3) 責めない UI を維持した催促なし設計、を新規 ADR で決める。MVP マトリクスは個人完結のままとする。
