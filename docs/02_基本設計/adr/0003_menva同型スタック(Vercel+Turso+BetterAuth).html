02 基本設計 · ADR-0003

ADR-0003 menva 同型スタック（Vercel + Turso + Better Auth）

おうち掃除ログ（sorosoro）の技術選定の正。menva-ai と同型スタックを採用する。

確定メモ

本 ADR は ADR-0001 および ADR-0002 を supersede する。設計書・実装は本 ADR のみを参照する。

ステータス

Accepted

日付

2026-07-20

背景

おうち掃除ログは B2C 個人向けの掃除記録アプリである。機種変更後もデータが残り、運用スキルを menva-ai と共有できることが重要になった。静的ファイルのみ・端末ローカル永続だけではバックアップとマルチデバイスに弱い。一方で社内業務向けの大規模基盤は過剰である。menva-ai と同型のクラウド Web スタックを採用し、ドメイン（責めない UI・1タップ記録・リスト型）に集中する。

決定

層 採用
フレームワークNext.js App Router（TypeScript）
ホスティングVercel（本番）/ next dev（ローカル）
DBTurso (libSQL) + Drizzle ORM + @libsql/client
ローカル DBturso dev --db-file local.db → http://127.0.0.1:8080
認証Better Auth + Google OAuth
OAuth 資格情報Google Cloud Console（GCP）
ミューテーションServer Actions（src/actions/tasks.ts）
スタイルTailwind CSS v4 + デザイントークン + Zen Maru Gothic
ドメインテーブルtasks（user_id, name, cycle_days）/ task_logs（task_id, done_date unique）
認証テーブルusers / sessions / accounts / verifications
 
Browser（モバイルファースト）
  → Next.js App Router on Vercel
       ├ Middleware: session cookie ゲート（/home）
       ├ Better Auth（/api/auth/*）→ Google OAuth（GCP）
       ├ Server Actions（tasks CRUD / record / undo）
       └ Drizzle → Turso libSQL

検討した代替案と却下理由

代替案却下理由
静的 Web + 端末内のみ永続（旧ローカルファースト案）機種変更・バックアップに弱い。クラウド前提の個人アプリ方針と不一致
ネイティブアプリWeb モバイルファーストで十分。配信・更新コストが高い（OUT-04）

影響・制約

  • Google ログインが必須。未認証はアプリ画面に入れない。
  • データはユーザー単位。世帯共有は MVP 外。
  • 環境変数は .env.example を正とする（Turso / Better Auth / GCP）。
  • prototype/ は UI 参照用。本番実装は src/。
  • UI 原則（責めない・1タップ・リスト型）とドメイン計算は企画書・PRODUCT_BRIEF どおり維持。
  • ADR-0001 / ADR-0002 は Historical / Superseded。実装ガイドとして使わない。
