テーブル定義

物理テーブルの定義と設計意図を記す。DB は PostgreSQL、スキーマ変更は Alembic マイグレーションを正とする。

記入ガイド

{{ }} を自アプリの値に置換。(例)テーブルは削除。

正は alembic/versions のマイグレーション。本書は意図の説明に徹する。

監査カラムは全テーブル共通のため下の共通節にまとめ、各表では省略する。

共通監査カラム: 下記は Base + TimestampMixin 継承の全テーブルに付与。各表定義では省略する。

テーブル定義雛形: テーブルごとに複製。監査カラムは記載不要。

インデックス設計: PK・UNIQUE 以外の索引を列挙。設定理由(絞り込み・整合性)を必ず書く。

正の所在

  • テーブルの正は alembic/versions/ のマイグレーションファイル。定義変更はマイグレーションで行い、本書は追随する。
  • モデル定義は app/models/models.py。マスタは m_、トランザクションは t_ プレフィックス。
  • 外部認証基盤(ap-safedx)参照テーブルは ExternalBase 継承で Alembic 対象外(本アプリからスキーマ変更しない)。
