## 1. 一言で言うと

家の掃除タスク（シーツ洗い、風呂の排水口など）の**実施履歴**と「**次の目安**」を管理する、**スマホ向け Web アプリ**。  
カレンダー型ではなく**リスト型**で、「前回から何日経過したか」だけを見せる。  
**サボりを責めない UI** が最重要コンセプト。記録は **1 タップ** で完結する。  
データは **ログインユーザー単位**で **Turso** に保存する。

---

## 2. コアドメインモデル

```
Task
├ id          : string (nanoid)
├ userId      : string（ログインユーザー）
├ name        : タスク名
├ cycleDays   : 目安周期（日数 1–365）
└ logs[]      : 実施日（TaskLog.doneDate "YYYY-MM-DD"、タスク内重複なし）

派生値（永続化しない。毎回計算）
├ lastDone     = logs の最新日（なければ null）
├ elapsedDays  = today - lastDone（日付単位。未記録時は null）
├ ratio        = logs 空 → 1.0 / それ以外 → elapsedDays / cycleDays
└ nextDue      = lastDone + cycleDays（未記録時は null）
```

### 並び順
1. `ratio` 降順  
2. 同率なら `elapsedDays` 降順（未記録は elapsed を最大扱い）

### 状態（色は補助のみ。警告アイコン・強い赤は禁止）
| 条件 | 状態 | 背景 |
|------|------|------|
| ratio ≥ 1.5 | `late` | late-bg |
| ratio > 1 | `warn` | warn-bg |
| それ以外 | 通常 | card |

### 記録ルール
- 「今日やった」→ 今日を logs に追記  
- 同日重複 → ブロック + トースト  
- 未来日は不可  
- 固定スケジュールなし

## 5. デザイン原則

1. **サボりを責めない**  
2. **記録は 1 タップ**  
3. **メモ帳のトーン** — Zen Maru Gothic、余白、淡色、フラット  

トークン: bg `#F6F2EA` / accent `#6FA378` / warn-bg `#F8EFD3` / late-bg `#F8E5DA` / del `#BC6248`

## 6. 技術スタック（正・ADR-0003）

menva-ai と同型。

| 層 | 採用 |
|----|------|
| フレームワーク | Next.js App Router (TypeScript) |
| ホスティング | Vercel（本番）/ `next dev`（ローカル） |
| DB | Turso (libSQL) + Drizzle ORM |
| ローカル DB | `turso dev --db-file local.db` → `http://127.0.0.1:8080` |
| 認証 | Better Auth + Google OAuth |
| OAuth 資格情報 | Google Cloud Console (GCP) |
| ミューテーション | Server Actions |
| スタイル | デザイントークン + Tailwind CSS v4 |

**実装パス**: リポジトリ root の `src/`。UI 参照用に `prototype/` あり（本番コードではない）。
